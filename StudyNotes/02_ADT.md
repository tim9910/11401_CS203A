## 抽象資料型態（ADT） 

### **詳細內容請參考老師提供的講義**： [01_Introduction.md](https://github.com/yfhuang/11401_CS203A/blob/main/StudyNote/01_Introduction.md)

### 重點整理

| 主題 | 重點內容 |
|---|---|
| ADT 定義 | 定義「資料＋可做的操作」，不討論實作細節。 |
| 為什麼需要 ADT？ | 使用者不用管內部怎麼做，實作可替換、程式更好維護。 |
| ADT vs 資料結構 | ADT 是**介面/規格**，資料結構是**實作方法**。 |
| 常見 ADT 例子 | Stack、Queue、List、Set 等 |
| STL 與 ADT 關係 | STL 直接提供 ADT 的介面，可用但不需理解內部細節。 |
| Boost 與 ADT | header 定義介面，cpp 定義實作 → 介面與實作分離。 |
